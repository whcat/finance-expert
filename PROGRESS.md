### 模型選用建議（2026-07-07 評估，日後執行時依此選模型）
> Claude Code 內用 `/model` 切換；定價為每百萬 token 輸入/輸出（2026-06 資料）。

**日常分工原則：**
- **單人節目例行萃取**（股癌、財經皓角）→ **Sonnet 5**（$3/$15，2026/8/31 前 $2/$10）：品質足夠、最省額度
- **多人訪談分離、profile 彙整、跨講者交叉比對** → **Opus 4.8**（$5/$25）：全部工作都能做、成本是 Fable 一半
- **最難任務**（三人以上快速交鋒、跨數十份觀點檔的深度比對）→ **Fable 5**（$10/$50）：最穩但最貴最慢，例行集數勿用
- **Haiku 4.5**（$1/$5）：❌ 不可用於核心萃取（語氣細讀、事實/〔推測〕區分會頻繁出錯），僅限週邊工具（別名比對、關鍵字打標）

**各模型能力邊界（辦不到的程度）：**
| 任務 | Haiku 4.5 | Sonnet 5 | Opus 4.8 | Fable 5 |
|---|---|---|---|---|
| 單人觀點萃取 | ⚠️ 勉強 | ✅ | ✅ | ✅ |
| 2–3 人語境切分（點名式訪談） | ❌ | ✅ | ✅ | ✅ |
| 3 人以上快速交鋒歸屬 | ❌ | ⚠️ 信心下降 | ⚠️ 較穩 | ⚠️ 最穩（仍非百分百） |
| 語氣細讀（戲謔/認真、條件式/肯定） | ❌ | ✅ 偶爾漏 | ✅ | ✅ |
| 言行落差偵測（未提示下主動發現） | ❌ | ⚠️ 需明確提示 | ✅ | ✅ |
| 跨數十份觀點檔 profile 彙整 | ❌（200K 上下文不足） | ✅ | ✅ | ✅ |
| Whisper 誤轉校正（冷門人名/黑話） | ⚠️ 常放過 | ✅ | ✅ | ✅ |

**四個模型共同的天花板（換模型無解）：**
1. 聽不到聲音——無講者標記時歸屬永遠是推理，要突破只能上語者分離（見講者分離方案二）
2. 救不回 Whisper 弄丟的內容（如卡帶段落）
3. 無法查證逐字稿內的數字（除非接網路搜尋）
4. 不能判斷觀點對錯——靠 profile 的「可驗證預測記錄表」讓時間驗證

**未來自動化參考**：改走 API 時預設 Sonnet 5＋Batch API（半價），單集萃取成本約 NT$3–8；多講者集數在腳本內升級 Opus 4.8。
